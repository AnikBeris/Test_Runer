## 📊 支持的架构与设备

<details> 
    <summary>⚙️ 展开描述</summary>

我们的平台支持广泛的架构与设备，以适应各种计算环境。以下是我们支持的主要架构：

- **amd64:** 本地用户和服务器的标准架构，支持现代大多数操作系统。

- **x86 / i386:** 广泛应用于桌面电脑和笔记本，有良好的系统和应用兼容性，包括 `Windows, macOS 和 Linux` 等。

- **armv8 / arm64 / aarch64:** 针对现代移动与嵌入式设备，如智能手机和平板电脑。设备实例：`Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS` 等。

- **armv7 / arm / arm32:** 应用于较旧的移动与嵌入式设备。依然流行，如 `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2` 等。

- **armv6 / arm / arm32:** 用于非常老旧的嵌入式设备，未完全普及但仍可见，典型设备：`Raspberry Pi 1, Raspberry Pi Zero/Zero W`。

- **armv5 / arm / arm32:** 较早期的嵌入式架构，今日稀有但仍用于部分旧设备，例如旧版 `Raspberry Pi` 和一些老款智能手机。

- **s390x:** `IBM` 主机架构，适用于企业任务负载。

</details>

## 📊 功能 || 特性

<details> 
    <summary>⚙️ 展开描述</summary>

- 系统状态监控
- 搜索所有入站连接和客户端
- 主题：`暗色 / 浅色`
- 支持多用户与多协议
- 支持协议包括：`VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- 支持本地 XTLS 协议：`RPRX-Direct, Vision, REALITY`
- 流量统计、流量限制、到期时间控制
- 可定制的 `Xray` 配置模板
- 支持 `HTTPS` 面板访问 (自定义域名+SSL 证书)
- 一键获取 `SSL 证书` 并自动续期
- 更高级配置选项请参见面板
- 提供修复的 `API 路由`(通过 `API` 创建用户设置)
- 通过面板支持设置不同参数的配置更改
- 支持通过面板导入/导出数据库

</details>

<div align="center">

管理脚本已内置通过 `Cloudflare` 请求 `SSL 证书` 的功能。要通过脚本获取证书，您需要：


<details> 
    <summary> ⚙️ 展开描述 </summary>

- 注册在 `Cloudflare` 的电子邮箱
- Cloudflare 全局 API 密钥
- 通过 `Cloudflare` 将您的域名指向当前服务器（DNS 配置）



## 如何获取 Cloudflare 全局 API 密钥：

1. 在终端中执行命令 `x-ui`，然后选择 `Cloudflare SSL Certificate`。

2. 访问以下链接：[Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击 `View Global API Key` (见下图):

</div>

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">

# 1. 快速安装 3X

<details> 
    <summary> ⚙️ 展开描述 </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

# 📊 安装必要的包

1. 更新系统
在开始安装之前，请确保系统已更新。执行以下命令：

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. 安装必要的包

确保您的服务器系统安装有以下必要包：

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## 安装 3X-UI 面板

要在您的服务器上安装面板，请运行以下脚本：

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

在安装过程中，脚本会询问：

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

输入 `y` 手动设置端口，或输入 `n` 让脚本自动决定。
避免使用常规端口 `22, 80, 8080`，建议选择任意其他端口，比如 `8181`。


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

完成上述步骤后，面板安装完成，可以通过以下路径在浏览器中访问该面板：

```sh
http://您的服务器IP地址:端口/WebBasePath (示例: http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

输入地址后，会进入面板登录界面，输入脚本之前提供的用户名和密码完成登录。

## 配置 DNS

<details> 
    <summary> ⚙️ 展开描述 </summary>


进入 `X-Ray 设置` -> 选择 `DNS` -> 找到子选项 `DNS`。

<div align="center">
  <img src="../media/Tutorial/Article_3/Setting_Xray_DNS.png" alt="设置 Xray DNS" width="70%">
</div>

点击 `创建 DNS`，逐一输入 DNS 地址：

```bash
tcp://8.8.8.8
```

```bash
tcp://1.1.1.1
```

```bash
tcp://9.9.9.9
```
