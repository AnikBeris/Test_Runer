## 📊 支持的架构与设备

<details> 
    <summary>⚙️ 展开详情</summary>

我们的平台兼容多种架构和设备，确保在不同计算环境中的灵活使用。支持的主要架构包括：

- **amd64:** 常见的个人电脑与服务器标准架构，支持绝大部分现代操作系统。

- **x86 / i386:** 广泛应用于台式电脑和笔记本。支持众多操作系统和应用程序，包括 `Windows, macOS 和 Linux`。

- **armv8 / arm64 / aarch64:** 适用于现代移动设备和嵌入式设备，如智能手机和平板电脑。设备示例：`Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`以及其他。

- **armv7 / arm / arm32:** 适用于较老的移动设备和嵌入式设备。仍广泛应用于如`Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`等设备。

- **armv6 / arm / arm32:** 针对非常旧的嵌入式设备。尽管已经较少见，仍被用于如`Raspberry Pi 1, Raspberry Pi Zero/Zero W`这样的设备中。

- **armv5 / arm / arm32:** 较老的架构，主要用于早期嵌入式系统。目前较为罕见，但可以用于旧设备，例如早期版本的`Raspberry Pi`及一些旧款智能手机。

- **s390x:** 此架构多数用于`IBM`主机，为企业工作负载提供了高性能和可靠性。

</details>

## 📊 功能 || 特性

<details> 
    <summary>⚙️ 展开详情</summary>

- 系统状态监控
- 所有传入连接和客户端的搜索
- 主题模式：`深色 / 浅色`
- 支持多用户和多协议
- 支持的协议：`VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- 原生 XTLS 协议支持：`RPRX-Direct, Vision, REALITY`
- 数据流量统计、流量限制、过期时间限制
- 可定制的`Xray`配置模板
- 支持通过`HTTPS (自定义域名+SSL证书)`访问控制面板
- 支持一键获取`SSL证书`和自动续期
- 更多高级配置选项请参阅控制面板
- 修复了`API路由`（用户设置可通过`API`生成）
- 支持在控制面板中通过不同参数更改配置
- 可通过控制面板导入/导出数据库

</details>

<h1 align="center">
Cloudflare 的 SSL 证书
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

管理脚本包含通过`Cloudflare`获取`SSL证书`的内置功能。要使用此脚本获取证书，您需具备：

<details> 
    <summary> ⚙️ 展开详情 </summary>

- 在 `Cloudflare` 注册的电子邮件
- Cloudflare 的 Global API Key
- 必须通过 `Cloudflare` 在 DNS 中将域名定向到当前服务器

### 如何获取 Global API Key Cloudflare：

1. 在终端中执行命令`x-ui`，然后选择`Cloudflare SSL Certificate`。

2. 点击链接：[Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击`View Global API Key`（参见下图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. 可能需要再次验证身份。验证后将显示 API 密钥（参见下图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

使用时，只需输入您的域名、`电子邮箱`和`API KEY`。示例如下：

<h1 align="center"> 
安装 3X-UI
</h1>

# 1. 快速安装 3X

<details> 
    <summary> ⚙️ 展开详情 </summary>

```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh
```

</details> 

# 2. 分步骤安装 3X

<details> 
    <summary> ⚙️ 展开详情 </summary>

# 📊 安装必要包

1. 系统更新
在安装前，请确保系统已完成更新。运行以下命令：

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. 安装必要的包

确保您的服务器已安装以下必需包：

```sh
apt-get install wget curl openssl qrencode systemd -y
```

## 安装 3X-UI 面板

在服务器上运行以下脚本完成面板安装：

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

安装过程中脚本会询问：

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

选择`y`即可手动设置端口，输入`n`脚本会自动设置端口。  
请勿使用`22, 80, 8080`等标准端口，建议选择其他任意端口，如`8181`。

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

完成这些步骤后，面板安装完成，您可通过以下地址访问面板：

```sh
http://服务器IP地址:端口/WebBasePath(例: http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

在浏览器输入地址后，进入面板登录界面，输入此前脚本提示的用户名和密码。
