## 📊 支持的架构与设备

<details> 
    <summary>⚙️ 展开详情</summary>

我们的平台支持多种架构和设备，适用不同的计算环境。下列为支持的主要架构：

- **amd64:** 主流的桌面和服务器架构，大多数现代操作系统都支持。

- **x86 / i386:** 被广泛应用于桌面，也支持包括 `Windows, macOS 和 Linux` 在内的多数操作系统。

- **armv8 / arm64 / aarch64:** 现代移动和嵌入式设备的标准选择（如 `Raspberry Pi 4`）。

- **armv7 / arm / arm32:** 广泛用于如 `Raspberry Pi 2` 等旧设备。

- **armv6 / arm / arm32:** 针对非常老的嵌入式设备，如 `Raspberry Pi Zero/Zero W`。

- **s390x:** 通常用于 IBM 主机，在企业负载任务中提供出色的性能与可靠性。

</details>

## 📊 功能 || 特性

<details> 
    <summary>⚙️ 展开详情</summary>

- 系统状态监控
- 检索所有入站连接和客户端
- 主题支持：`暗模式/亮模式`
- 多用户及多协议支持
- 支持协议列表: `VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- 支持原生 XTLS 协议：`RPRX-Direct, Vision, REALITY`
- 流量统计、限制流量、时间过期限制
- `Xray` 可定制的配置模板
- 支持通过 `HTTPS` 访问控制面板（自定义域名 + SSL 证书）
- 一键请求 `SSL 证书` 并支持自动续订
- 更多高级配置参数请查看管理面板
- 改善 `API 路由`
- 导出/导入数据库的支持

</details>

<h1 align="center"> 
Cloudflare SSL证书
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

该管理脚本内置从 `Cloudflare` 获取 `SSL证书` 的功能。获取证书需要以下条件：

<details> 
    <summary> ⚙️ 展开详情 </summary>

- 注册 Cloudflare 的邮箱
- Global API Key
- 域名需通过 `Cloudflare` DNS 指向当前服务器

## 如何获取 Cloudflare 的 Global API Key:

1. 运行命令 `x-ui`, 然后选择 `Cloudflare SSL Certificate`.

2. 打开链接: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击 `View Global API Key` (参考下图):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. 验证身份后即可查看Key (参考下图):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

按提示输入域名、`email` 和 `API Key`。例如:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>
