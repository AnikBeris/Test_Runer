## 📊 支持的架构及设备

<details> 
    <summary>⚙️ 展开说明</summary>

平台兼容多种架构和设备，在不同计算环境中灵活使用。以下列出主要支持的架构：

- **amd64:** 普遍应用于个人电脑和服务器，是大部分现代操作系统的标准。

- **x86 / i386:** 常见于台式机和笔记本。广泛支持多类操作系统和应用程序，包括 `Windows、macOS 和 Linux` 等。

- **armv8 / arm64 / aarch64:** 针对现代移动和嵌入式设备，如智能手机和平板电脑。包括设备如 `Raspberry Pi 4、Raspberry Pi 3、Raspberry Pi Zero 2/Zero 2 W`、`Orange Pi 3 LTS` 等。

- **armv7 / arm / arm32:** 面向较旧的移动和嵌入式设备。仍然广泛应用于 `Orange Pi Zero LTS、Orange Pi PC Plus、Raspberry Pi 2` 等设备。

- **armv6 / arm / arm32:** 支持非常老旧的嵌入式设备。尽管其使用率较低，仍适用于如 `Raspberry Pi 1、Raspberry Pi Zero/Zero W`。

- **armv5 / arm / arm32:** 较旧架构，主要与早期嵌入式系统相关。如今已不常见，但可用于老旧设备如早期 `Raspberry Pi` 以及部分旧智慧手机。

- **s390x:** 通常用于 `IBM` 大型机，提供企业级工作负载所需的高性能和可靠性。

</details>

## 📊 功能特色

<details> 
    <summary>⚙️ 展开说明</summary>

- 系统状态监视
- 所有入站连接及客户端搜索功能
- 主题：`暗色主题/亮色主题`
- 支持多用户及多协议
- 支持协议：`VMESS、VLESS、Trojan、Shadowsocks、Dokodemo-door、Socks、HTTP、WireGuard`
- 支持本地 `XTLS` 协议：`RPRX-Direct、Vision、REALITY`
- 流量统计、流量限制、到期时间限制
- 可定制的 `Xray` 配置模板
- 支持以 `HTTPS（自定义域名 + SSL证书）` 访问面板
- 支持一键请求 `SSL证书` 并自动续期
- 其他高级设置可通过面板配置
- 修正的 `API路由`（用户设置通过 `API` 创建）
- 支持通过面板根据不同参数修改配置
- 支持通过面板导入/导出数据库

</details>

Cloudflare 的 SSL证书
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt="Cloudflare" width="50%">

管理脚本包含通过 `Cloudflare` 请求 `SSL证书` 的内置功能。使用该脚本获取证书需要满足以下条件：


<details> 
    <summary> ⚙️ 展开说明 </summary>

- 在 `Cloudflare` 注册的邮箱
- Global API Key Cloudflare
- 域名需通过 `Cloudflare` 的 DNS 指向当前服务器



## 如何获取 Global API Key Cloudflare：

1. 在终端执行命令 `x-ui`，然后选择 `Cloudflare SSL Certificate`。

2. 打开以下链接：[Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击 `View Global API Key`（如下图所示）：

</div>

# 1. 快速安装 3X

<details> 
    <summary> ⚙️ 展开说明 </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. 分步安装 3X


<details> 
    <summary> ⚙️ 展开说明 </summary>

# 📊 安装所需依赖项

1. 系统更新
在安装前，请确保系统已更新。执行以下命令：

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. 安装必要的软件包

确保您的服务器上已安装以下软件包：

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## 安装 3X-UI 面板

在服务器上运行以下脚本命令以安装面板：

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

在安装过程中脚本会提示您：

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

选择 `y` 来手动设置端口，或选择 `n` 让脚本随机设置。
请避免选择标准端口 `22, 80, 8080`，建议选择其他端口，如 `8181`。

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

完成以上步骤后，面板安装完成。使用服务器 IP 地址和设定的端口通过浏览器连接到控制面板。

```sh
http://服务器_IP地址:端口/WebBasePath（例如：http://192.168.0.10:40608/vpkPI6ex9ajesDX）
```

在浏览器中输入地址后，会进入面板登录页面。在此输入脚本安装时提供的登录用户名和密码。
