## 📊支持的平台架构与设备

<details> 
    <summary>⚙️展开描述</summary>

我们的平台支持多种常见架构和设备，以满足多样的计算环境需求。以下是主要支持的架构：

- **amd64:** 这种流行的架构是桌面电脑与服务器的标准架构，可以无缝运行大多数现代操作系统。

- **x86 / i386:** 广泛应用于桌面与笔记本电脑。此架构被多种操作系统和应用广泛支持，包括`Windows, macOS 和 Linux`。

- **armv8 / arm64 / aarch64:** 专用于现代移动及嵌入式设备，例如智能手机与平板。支持设备举例：`Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`等。

- **armv7 / arm / arm32:** 适用于较旧的移动与嵌入式设备。仍被广泛应用于例如`Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`等设备。

- **armv6 / arm / arm32:** 针对年较久远的嵌入式设备。虽然使用较少，但仍有应用场景，例如`Raspberry Pi 1, Raspberry Pi Zero/Zero W`。

- **armv5 / arm / arm32:** 更老的架构，主要用于较早期的嵌入式系统。虽目前少见，但适配于旧设备，如早期版本的`Raspberry Pi`和一些老款智能手机。

- **s390x:** 该架构多用于`IBM`的大型机，提供企业级工作负载的高性能与可靠性。

</details>

## 📊功能特点

<details> 
    <summary>⚙️展开描述</summary>

- 系统状态监控
- 全部入站连接及客户端搜索

- 修复后的 `API路由`（用户设置通过 `API` 创建）
- 支持通过面板对多项参数的配置修改
- 数据库导出/导入支持（面板操作）

<h1 align="center"> 
Cloudflare 的 SSL 证书
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

管理脚本内置通过 `Cloudflare` 请求 `SSL证书` 的功能。通过此脚本获取证书需要以下条件：


<details> 
    <summary> ⚙️展开描述 </summary>

- 注册 `Cloudflare` 的邮箱
- Cloudflare 的 Global API Key
- 域名需通过 `Cloudflare` DNS 指向当前服务器



## 如何获取 Cloudflare 的 Global API Key：

1. 在终端中运行命令 `x-ui`，然后选择 `Cloudflare SSL证书`。

2. 打开链接：[Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击 `View Global API Key`（参考下图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. 可能需要重新认证。认证通过后可以看到 API Key（参考下图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

<h1 align="center"> 
安装 3X-UI
</h1>


# 1. 快速安装 3X

<details> 
    <summary> ⚙️展开描述 </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. 分步安装 3X


<details> 
    <summary> ⚙️展开描述 </summary>

# 📊安装所需包

1. 更新系统
在安装前确保您的系统已经更新，输入以下命令：

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. 安装所需包

确保您的服务器安装了相关必要包：

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## 安装 3X-UI 面板

在服务器中运行以下脚本命令完成面板安装：

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

安装过程会提示以下问题：

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

若需要手动指定端口输入 `y`，否则输入 `n`脚本会随机分配端口。
请勿使用默认端口 `22, 80, 8080`，建议选择其他端口如 `8181`。


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

完成以上步骤后，面板安装完成，可通过以下路径连接到面板：

```sh
http://您的服务器IP地址:端口/WebBasePath(示例：http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

输入地址后会进入面板登录界面，需在此界面输入脚本步骤中确认的用户名及密码。
