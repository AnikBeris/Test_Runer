## 📊 支持的架构和设备

<details> 
    <summary>⚙️ 展开内容</summary>

我们的平台兼容范围广泛的架构和设备，为各种计算环境提供灵活性。以下是我们支持的主要架构：

- **amd64:** 这是个人计算机和服务器的标准架构，支持大多数现代操作系统的流畅运行。

- **x86 / i386:** 广泛应用于台式机和笔记本电脑中。该架构具有多个操作系统和应用程序的广泛支持，包括`Windows, macOS 和 Linux`等。

- **armv8 / arm64 / aarch64:** 为现代移动和嵌入式设备（如智能手机和平板电脑）设计的架构。示例设备有：`Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`等。

- **armv7 / arm / arm32:** 用于旧版移动和嵌入式设备架构。其中仍广泛应用于设备如`Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`等。

- **armv6 / arm / arm32:** 面向旧式嵌入式设备的架构。虽然少见，但仍应用于`Raspberry Pi 1, Raspberry Pi Zero/Zero W`之类的设备。

- **armv5 / arm / arm32:** 早期嵌入式系统的架构。尽管现在很少见，仍可能用于老旧设备，如早期版本`Raspberry Pi`或一些较老的智能手机中。

- **s390x:** 通常用于`IBM`主机中，为企业级负载提供高性能和可靠性。

</details>

## 📊 功能 || 特性

<details> 
    <summary>⚙️ 展开内容</summary>

- 系统状态监控
- 搜索所有入站连接和客户端
- 主题支持：`深色 / 浅色`
- 支持多用户和多协议
- 支持以下协议：`VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- 支持原生 XTLS 协议：`RPRX-Direct, Vision, REALITY`
- 流量统计、流量限制、过期时间限制
- 可配置的`Xray`配置模板
- 支持通过`HTTPS (自定义域名+SSL证书)`访问面板
- 支持一键获取 `SSL证书` 和其自动续期
- 面板提供更高级别的配置参数
- 修复`API端点` (通过`API`创建用户设置)
- 提供支持面板上基于不同参数的配置更改
- 支持在面板上导出/导入数据库

</details>

Cloudflare SSL证书
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

管理脚本包括通过`Cloudflare`获取`SSL证书`的功能。要通过此脚本获取证书，您需要：

<details> 
    <summary> ⚙️ 展开内容 </summary>

- 在`Cloudflare`注册的邮箱
- Cloudflare Global API Key
- 通过`Cloudflare`将域名解析（DNS解析）至当前服务器地址

## 如何获取Cloudflare Global API Key:

1. 在终端中运行命令`x-ui`，然后选择 `Cloudflare SSL Certificate`。

2. 打开链接：[Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击`View Global API Key`（详见下面截图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. 可能需要重新身份验证。验证成功后将显示API密钥（参考下图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

将您的域名、`email`和`API KEY`输入即可。如下示例所示：

<h1 align="center"> 
安装3X-UI
</h1>

# 1. 快速安装3X

<details> 
    <summary> ⚙️ 展开内容 </summary>

```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh
```

</details> 

# 2. 分步骤安装3X

<details> 
    <summary> ⚙️ 展开内容 </summary>

# 📊 必要组件安装

1. 更新系统
开始安装前，请确保系统已更新。执行以下命令：

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. 安装必需包

确保以下必要的组件已安装至您的服务器上：

```sh
apt-get install wget curl openssl qrencode systemd -y
```

## 安装3X-UI面板

运行以下脚本以安装面板至您的服务器：

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

安装时脚本会询问：

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

回答`y`可自行设置端口，选择`n`则由脚本随机分配端口。
不要选择默认端口，如`22, 80, 8080`等，推荐使用如`8181`的非默认端口。

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

安装完成后，可以通过浏览器访问该路径进行面板登录

```sh
http://您的服务器IP地址:端口/WebBasePath (示例: http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

访问后，将进入登录界面，请输入脚本安装完成时提示的管理员用户名称和密码。
