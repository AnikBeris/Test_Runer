## 📊 支持的架构与设备

<details> 
    <summary>⚙️ 展开说明</summary>

我们的平台兼容广泛的系统架构和设备，确保在不同的计算环境中具有灵活性。以下是主要支持的架构：

- **amd64:** 这种常见架构是个人电脑和服务器的标准，支持大多数现代操作系统的流畅运行。

- **x86 / i386:** 广泛用于台式计算机和笔记本电脑，支持多个操作系统和应用程序，如`Windows, macOS 和 Linux`。

- **armv8 / arm64 / aarch64:** 针对现代移动和嵌入式设备，例如智能手机和平板电脑。示例设备包括`Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`等。

- **armv7 / arm / arm32:** 用于较老的移动和嵌入式设备，仍然广泛应用于设备如`Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`等。

- **armv6 / arm / arm32:** 主要面向非常老旧的嵌入式设备，仍能见于如`Raspberry Pi 1, Raspberry Pi Zero/Zero W`等。

- **armv5 / arm / arm32:** 是早期嵌入式系统使用的一种更老旧的架构，现在已较为罕见，但在一些老旧设备如早期`Raspberry Pi`以及某些旧智能手机中仍可见。

- **s390x:** 一种常见于IBM大型机的架构，能够为企业负载提供高性能和可靠性。

</details>

## 📊 功能 || 特点

<details> 
    <summary>⚙️ 展开说明</summary>

- 系统状态监控
- 搜索所有入站连接和客户端
- 主题模式：`暗黑 / 明亮`
- 支持多用户和多协议
- 支持的协议：`VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- 支持原生的XTLS协议：`RPRX-Direct, Vision, REALITY`
- 流量统计、流量限制、过期时间限制
- 可定制的`Xray`配置模板
- 支持通过`HTTPS（自定义域名 + SSL证书）`访问面板
- 支持一键申请`SSL证书`及其自动续期
- 对复杂配置参数请参考面板
- 修复后的`API路由`（用户设置通过`API`创建）
- 面板支持的多种参数下更改配置
- 面板支持数据库的导出/导入

</details>

</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">

管理脚本包含通过`Cloudflare`请求`SSL证书`的内置功能。使用此脚本获取证书需要以下条件：


<details> 
    <summary> ⚙️ 展开说明 </summary>

- 在`Cloudflare`中注册的电子邮件地址
- Cloudflare的Global API Key
- 域名必须通过`Cloudflare`的DNS指向当前服务器



## 如何获得Cloudflare的Global API Key：

1. 在终端中执行命令`x-ui`，选择`Cloudflare SSL Certificate`。

2. 打开以下链接：[Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击`View Global API Key`（参见下图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. 可能需要再次验证身份，之后将展示Key（参见下图）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

<h1 align="center"> 
安装 3X-UI
</h1>


# 1. 快速安装 3X

<details> 
    <summary> ⚙️ 展开说明 </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. 手动安装 3X


<details> 
    <summary> ⚙️ 展开说明 </summary>

# 📊 安装所需包

1. 更新系统
在开始安装前，请确保您的系统是最新的。执行以下命令：

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. 安装必要的包

确保您已在服务器上安装以下必要包：

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## 安装 3X-UI 面板

在服务器上运行以下命令安装面板：

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

安装过程中脚本将会询问：

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

请输入`y`以自定义配置端口，或选择`n`让脚本随机生成端口。
请不要选择标准端口如`22, 80, 8080`，建议选择其他端口，例如`8181`。


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

完成这些步骤后，面板安装完成。可以通过浏览器使用以下路径访问面板：

```sh
http://您的服务器IP地址:端口/WebBasePath(例如：http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

在浏览器中输入地址后将进入登录界面，输入脚本先前提供的用户名和密码以登录面板。
