## 📊 支持的架构和设备

<details> 
    <summary>⚙️ 展开详细说明</summary>

我们的平台具备广泛的架构和设备兼容性，能够灵活应用于各种计算环境。以下是主要支持的架构列表：

- **amd64:** 主流PC和服务器使用的标准架构，能够流畅运行大多数现代操作系统。

- **x86 / i386:** 常用于台式机和笔记本电脑，兼容多种操作系统和应用程序，包括 `Windows, macOS 和 Linux`。

- **armv8 / arm64 / aarch64:** 面向现代移动设备和嵌入式设备，例如 `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS` 等。

- **armv7 / arm / arm32:** 用于较早的移动和嵌入式设备，适用于如 `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2` 等设备。

- **armv6 / arm / arm32:** 针对非常老旧的嵌入式设备，例如 `Raspberry Pi 1, Raspberry Pi Zero/Zero W`。

- **armv5 / arm / arm32:** 面向早期嵌入式系统，目前较少使用，但可能出现在如一些旧设备中。

- **s390x:** 通常用于 `IBM` 主机，应对企业级使用环境的高性能和稳定需求。

</details>

## 📊 功能集 || 特点

<details> 
    <summary>⚙️ 展开详细说明</summary>

- 系统状态监控
- 全入网连接和客户端搜索
- 主题支持：`深色模式 / 浅色模式`
- 支持多个用户和多协议
- 支持协议：`VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- 支持原生 XTLS 协议：`RPRX-Direct, Vision, REALITY`
- 流量统计、流量限制、到期时间限制
- 可自定义配置模板 `Xray`
- 支持通过 `HTTPS`（自定义域名 + SSL 证书）访问面板
- 支持一键申请 `SSL 证书`及其自动续期
- 更高级的配置选项请参考面板
- 修复的 `API 路由`（通过 API 设置用户配置）
- 支持通过面板根据不同参数修改配置
- 支持通过面板导入/导出数据库

</details>

<h1 align="center"> 
Cloudflare 的 SSL 证书
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

管理脚本集成了通过 `Cloudflare` 获取 `SSL 证书` 功能。想要使用脚本获取证书，您需要：

<details> 
    <summary> ⚙️ 展开说明 </summary>

- 一个注册在 `Cloudflare` 上的电子邮箱
- Cloudflare 的 Global API Key
- 一个已通过 `Cloudflare` DNS 指向当前服务器的域名

## 如何获取 Cloudflare 的 Global API Key：

1. 在终端输入命令 `x-ui`，然后选择 `Cloudflare SSL Certificate`。

2. 打开以下页面获得 API 密钥：[Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. 点击 `View Global API Key`（下图展示）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. 系统可能会要求重新验证身份，之后会显示密钥（下图展示）：

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

在使用过程中，只需输入您的域名，`邮箱` 和 `API KEY`。以下截图为操作示例：

<h1 align="center"> 
安装 3X-UI
</h1>

# 1. 快速安装 3X

<details> 
    <summary> ⚙️ 展开详解 </summary>

```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh
```

</details> 

# 2. 分步安装 3X

<details> 
    <summary> ⚙️ 展开详解 </summary>

# 📊 安装必要包

1. 更新系统
在安装前，请确保系统已更新，运行以下命令：

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. 安装必需的包

检查服务器是否已安装以下插件：

```sh
apt-get install wget curl openssl qrencode systemd -y
```

## 安装 3X-UI 面板

通过以下脚本在服务器上安装面板：

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

在安装过程中，脚本会询问以下问题：

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

回答 `y`，以自定义端口，或回答 `n`，让脚本随机选择端口。
请勿使用 `22, 80, 8080` 等常见端口，建议选择其他端口，如 `8181`。

安装后面板将提供以下信息：

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

完成安装后，可以通过浏览器访问以下 URL：

```sh
http://服务器的IP地址:端口/WebBasePath（例如：http://192.168.0.10:40608/vpkPI6ex9ajesDX）
```

在浏览器中输入该地址后，登录页面将显示，使用安装时提供的用户名和密码登录。

## 配置 DNS

<details> 
    <summary> ⚙️ 展开详解 </summary>

<div align="center">
