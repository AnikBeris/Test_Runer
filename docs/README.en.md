## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform offers compatibility with a wide range of architectures and devices, ensuring flexibility across diverse computing environments. Below are the main architectures we support:

- **amd64:** This common architecture is a standard for personal computers and servers, ensuring seamless operation for most modern operating systems.

- **x86 / i386:** Widely used in desktops and laptops. The architecture has strong support from numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices like smartphones and tablets. Examples of supported devices include `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Serves as an architecture for older mobile and embedded devices. It is still widely used in devices such as `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and others.

- **armv6 / arm / arm32:** Targets much older embedded devices. While less common, it is still utilized in devices like `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture mainly associated with early embedded systems. It is now less common but can still be found in outdated devices like early versions of `Raspberry Pi` and some older smartphones.

- **s390x:** This architecture is typically used in IBM mainframes and provides high performance and reliability for enterprise workloads. 

</details>

## üìä Features

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

- System state monitoring
- Search through all incoming connections and clients
- Theme support: `dark/light`
- Multiple user and multi-protocol support

- Support for one-click `SSL certificate` requests and automatic renewal
- Advanced configuration options available via the panel
- Fixed `API routes` (user settings are created via `API`)
- Database export/import support via the panel

</details>

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in function to request an `SSL certificate` via `Cloudflare`. To obtain the certificate using this script, you need:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- An email registered with `Cloudflare`
- Global API Key from Cloudflare
- The domain name must be pointed (configured in DNS) to the current server via `Cloudflare`



## How to Obtain your Cloudflare Global API Key:

1. Execute the command `x-ui` in the terminal, then select `Cloudflare SSL Certificate`.

2. Visit the following link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. Re-authentication may be required. Once done, the key will be displayed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


When using, simply enter your domain, `email`, and `API Key`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

<h1 align="center"> 
Installing 3X-UI
</h1>


# 1. Quick Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-Step Installation of 3X


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Required Packages

1. System Update
Before starting the installation, ensure your system is updated. Execute the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Installing Required Packages

Ensure that the necessary packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing the 3X-UI Panel

To install the panel on the server, execute the script using the following command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will ask the question:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer `y` if you want to set the port yourself, or `n` if you prefer the script to set the port automatically.
Do not use the standard ports `22, 80, 8080`. Using a port like `8181` is recommended.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the panel installation process will be complete, and you can access the panel via a browser using the URL:

```sh
http://YOUR_SERVER_IP:Port/WebBasePath (example: http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

Once the address is entered in the browser, you'll be directed to the login screen, where you'll need to input the username and password that were generated by the script.
