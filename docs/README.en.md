## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Description</summary>

Our platform supports a wide range of architectures and devices, providing flexibility for various computational environments. Below is a list of the major architectures supported:

- **amd64:** This common architecture is standard for personal computers and servers, ensuring seamless operation of most modern operating systems.

- **x86 / i386:** Widely used in desktop computers and laptops, with extensive support from numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices, such as smartphones and tablets. Examples: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Serves as the architecture for older mobile and embedded devices. Still widely used in devices like `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and others.

- **armv6 / arm / arm32:** Geared towards very old embedded devices. While less common, still used in `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** Older architecture mainly associated with early embedded systems. Rarely encountered today but may be found in legacy devices like early `Raspberry Pi` models and some older smartphones.

- **s390x:** Typically used in `IBM` mainframes, offering high performance and reliability for enterprise workloads.

</details>

## üìä Functions || Features

<details> 
    <summary>‚öôÔ∏è Description</summary>

- System status monitoring
- Search across all incoming connections and clients
- Theme: `dark/light`
- Support for multiple users and multi-protocols
- Supported protocols: `VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- Native XTLS protocol support: `RPRX-Direct, Vision, REALITY`
- Traffic statistics, traffic limits, and expiration time control
- Configurable `Xray` configuration templates
- Support for HTTPS (custom domain + SSL certificate) access to the panel
- Support for one-click requests for `SSL certificates` and their automatic renewal
- For more advanced configuration parameters, see the panel
- Fixed `API routes` (user configurations created via `API`)
- Modification support for configurations based on available panel parameters
- Database export/import support via the panel

<h1 align="center"> 
SSL Certificate with Cloudflare
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt="Cloud flare" width="50%">
</div>

The management script includes a built-in function to request an `SSL certificate` through `Cloudflare`. To obtain the certificate using this script, you need to:

<details> 
    <summary> ‚öôÔ∏è Description </summary>

- Email registered with `Cloudflare`
- Cloudflare Global API Key
- Domain name must be pointed (set in DNS) to the current server via `Cloudflare`.

## How to get a Cloudflare Global API Key:

1. In the terminal, execute the `x-ui` command, then select `Cloudflare SSL Certificate`.

2. Follow this link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click on `View Global API Key` (see the screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt="API Key 1" width="70%">
</div>

4. You may be asked to authenticate again. The key will then be displayed (see the screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt="API Key 2" width="70%">
</div>

When using, simply enter your domain, `email`, and `API KEY`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt="Detail Enter" width="70%">
</div>

</details>

<h1 align="center"> 
3X-UI Installation
</h1>

# 1. Quick 3X Installation

<details> 
    <summary> ‚öôÔ∏è Details </summary>

```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh
```

</details> 

# 2. Step-by-Step 3X Installation

<details> 
    <summary> ‚öôÔ∏è Details </summary>

# üìä Installing Required Packages

1. System Update
Before starting the installation, ensure your system is up to date. Execute the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Install the Required Packages

Make sure the necessary packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```

## Installing 3X-UI Panel

To install the panel on the server, run the following script:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During installation, the script will ask:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Respond with `y` to set the port manually, or `n` for random. Avoid using standard ports like `22, 80, 8080`. A recommended port could be `8181`.

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the panel installation will be finalized. You can connect through your browser using the following format:

```sh
http://SERVER_IP_ADDRESS:PORT/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

Use your browser to access the panel. Log in with the credentials provided by the script.
