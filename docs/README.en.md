## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform offers compatibility with a wide range of architectures and devices, ensuring flexibility across various computational environments. Below are the main supported architectures:

- **amd64:** This common architecture is the standard for personal computers and servers, enabling smooth operation of most modern operating systems.

- **x86 / i386:** Widely used in desktop computers and laptops. This architecture has broad support from numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Specifically designed for modern mobile and embedded devices, such as smartphones and tablets. Examples of devices: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Serves as the architecture for older mobile and embedded devices. Still widely used in devices such as `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and others.

- **armv6 / arm / arm32:** Focused on very old embedded devices, although less prevalent, it is still utilized in `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture mainly associated with early embedded systems. Rarely encountered today, it can be found in legacy devices such as early versions of `Raspberry Pi` and older smartphones.

- **s390x:** Typically used in IBM mainframes, providing high performance and reliability for enterprise workloads.

</details>

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloudflare " width="50%">
</div>

The management script includes a built-in function to request an `SSL certificate` via `Cloudflare`. To obtain the certificate using this script, you need:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- An email registered with `Cloudflare`
- Global API Key from Cloudflare
- Domain name should be pointed (defined in DNS) to the current server through `Cloudflare`



## How to obtain the Global API Key from Cloudflare:

1. In the terminal, execute the command `x-ui`, then select `Cloudflare SSL Certificate`.

2. Open the link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. Reauthentication may be required. Once completed, the key will be revealed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


During usage, simply enter your domain, `email`, and `API KEY`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

</details>

# 1. Quick Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-step Installation of 3X


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Necessary Packages

1. System Update
Make sure your system is updated before starting the installation. Execute the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Installing Essential Packages

Ensure the required packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing the 3X-UI Panel

To install the panel on your server, execute the script command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will prompt:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Respond with `y` if you want to set the port manually, or `n`, and the script will select a port automatically.
Avoid using standard ports like `22, 80, 8080`. It's recommended to choose another, such as `8181`.

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

Following these steps, the panel installation will be complete, and you can connect via browser to the panel at:

```sh
http://Your_Server_IP:Port/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

After entering the address in your browser, you‚Äôll reach the panel login window where you‚Äôll need to input the provided username and password.
