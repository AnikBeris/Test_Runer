## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform offers compatibility with a wide range of architectures and devices, ensuring flexibility in various computing environments. Below are the key architectures we support:

- **amd64:** This widespread architecture is the standard for personal computers and servers, providing seamless operation for most modern operating systems.

- **x86 / i386:** Widely used in desktop computers and laptops. This architecture is extensively supported by numerous operating systems and applications, including, among others, `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices, such as smartphones and tablets. Examples of devices: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Serves as the architecture for older mobile and embedded devices. It is still widely used in devices like `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and others.

- **armv6 / arm / arm32:** Targeted at very old embedded devices. Although less common, it is still used, for example, in `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** A more outdated architecture mainly associated with early embedded systems. Today, it is rare but can be used in legacy devices such as older versions of `Raspberry Pi` and some early smartphones.

- **s390x:** This architecture is typically utilized in `IBM` mainframes and provides high performance and reliability for enterprise workloads.

</details>

## üìä Features || Capabilities

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

- Ability to modify configurations using various options available in the panel

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in function to request an `SSL certificate` through `Cloudflare`. To obtain a certificate using this script, you need to:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- Email registered in `Cloudflare`
- Global API Key from Cloudflare
- The domain name must be directed (registered in DNS) to the current server via `Cloudflare`



## How to get the Global API Key from Cloudflare:

1. In the terminal, execute the command `x-ui`, then select `Cloudflare SSL Certificate`.

2. Go to: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (refer to the screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. Re-authentication may be required. After that, the key will be displayed (refer to the screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


Simply enter your domain, `email`, and `API KEY` for use. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

</details>

<h1 align="center"> 
Installing 3X-UI
</h1>


# 1. Quick Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-step Installation of 3X


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Required Packages

1. System Update
Ensure your system is updated before starting the installation. Execute the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Installing Necessary Packages

Confirm that the required packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing the 3X-UI Panel

To install the panel on the server, run the script using the command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will ask:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer `y` to set the port yourself or `n` to let the script assign the port automatically. 
Do not use standard ports `22, 80, 8080`. Choosing a different one, for example, `8181`, is recommended.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the panel installation will be finished, and you can access it through the browser at:

```sh
http://Your_Server_IP_Address:Port/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

When you enter the address in your browser, you'll see the login window. Enter the username and password provided by the script earlier.
