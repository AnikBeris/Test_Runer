## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform is compatible with a vast range of architectures and devices, ensuring flexibility across various computing environments. Below are the main supported architectures:

- **amd64:** This common architecture is standard for personal computers and servers, ensuring seamless operation of most modern operating systems.

- **x86 / i386:** Widely used in desktops and laptops. Supported by numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices like smartphones and tablets. Examples: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, among others.

- **armv7 / arm / arm32:** Serves as the architecture for older mobile and embedded devices, still widely used in devices like `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and more.

- **armv6 / arm / arm32:** Targets very old embedded devices. While less common, it's still utilized in devices like `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture primarily tied to early embedded systems. It may be found in obsolete devices like early versions of `Raspberry Pi` and some old smartphones.

- **s390x:** Typically used in IBM mainframes, offering high performance and reliability for enterprise workloads.

</details>

## üìä Features || Capabilities

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

- System status monitoring
- Search across all incoming connections and clients
- Theme: `Dark/Light`
- Supports multi-user and multi-protocol

- For more advanced configuration options, refer to the panel
- Fixed `API routes` (user settings are configured via `API`)
- Support for varying configuration adjustments through panel options
- Export/import database through the panel

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloudflare " width="50%">
</div>

The management script comes with a built-in function for requesting an `SSL certificate` via `Cloudflare`. To obtain a certificate using this script, you need to:

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- An email registered with `Cloudflare`
- Global API Key from Cloudflare
- The domain name must be pointed (configured in DNS) to the current server via `Cloudflare`

## How to Get Cloudflare Global API Key:

1. Run the command `x-ui` in the terminal, then select `Cloudflare SSL Certificate`.

2. Visit the link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click on `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. You may be required to re-authenticate. After that, the key will be displayed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

When prompted, simply input your domain, `email`, and `API KEY`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

</details>

<h1 align="center"> 
3X-UI Installation
</h1>

# 1. Quick Install for 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh
```

</details> 

# 2. Step-by-Step Installation for 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Required Packages

1. System Update
Before starting, ensure your system is up-to-date. Run the following:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Install Required Packages

Ensure the necessary packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```

## Installing the 3X-UI Panel

To install the panel on the server, run the script using this command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During installation, you will be prompted:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer `y` if you wish to set the port manually or `n` to let the script randomly assign one.
Avoid standard ports such as `22, 80, 8080`. Instead, choose something like `8181`.

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

Once these steps are complete, the panel installation will finish, and you can access it via your browser.

```sh
http://YOUR_SERVER_IP:PORT/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

On the login screen, input the username and password provided by the script.
