## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand description</summary>

Our platform offers compatibility with a wide range of architectures and devices, providing flexibility in various computing environments. Below are the main supported architectures:

- **amd64:** This common architecture is standard for personal computers and servers, ensuring seamless operation for most modern operating systems.

- **x86 / i386:** Widely used in desktops and laptops, this architecture enjoys broad support from numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Aimed at modern mobile and embedded devices such as smartphones and tablets. Examples: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Serves as the architecture for older mobile and embedded devices. Still widely used in devices like `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and more.

- **armv6 / arm / arm32:** Designed for very old embedded devices, albeit less common. Used, for example, in `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture mostly associated with early embedded systems. Now rarely encountered, but still used in legacy devices like early `Raspberry Pi` versions and some old smartphones.

- **s390x:** This architecture is typically employed in `IBM` mainframes, offering high performance and reliability for enterprise workloads.

</details>

## üìä Features || Capabilities

<details> 
    <summary>‚öôÔ∏è Expand description</summary>

- System status monitoring
- Search across all incoming connections and clients
- Theme: `dark / light`
- Multi-user and multi-protocol support

- One-click `SSL certificate` request and automatic renewal support
- For advanced configuration parameters, refer to the panel
- Fixed `API routes` (user settings are created via `API`)
- Support for modifying configurations using various parameters available on the panel
- Export/Import database via panel support

</details>

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in function for requesting an `SSL certificate` via `Cloudflare`. To obtain a certificate using this script, you need:


<details> 
    <summary> ‚öôÔ∏è Expand description </summary>

- An email registered with `Cloudflare`
- Cloudflare Global API Key
- The domain name must be pointed (set in DNS) to the current server via `Cloudflare`



## How to obtain Cloudflare Global API Key:

1. Execute the command `x-ui` in the terminal, then select `Cloudflare SSL Certificate`.

2. Navigate to: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. Re-authentication might be required. After that, the key will be displayed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


When using, simply enter your domain, `email`, and `API KEY`. Example shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

</details>

<h1 align="center"> 
Installing 3X-UI
</h1>


# 1. Quick Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-Step Installation of 3X


<details> 
    <summary> ‚öôÔ∏è Expand description </summary>

# üìä Installing Required Packages

1. Update the system:
Before beginning the installation, ensure your system is up to date. Run the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Install required packages:

Ensure your server has the necessary packages installed:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Install the 3X-UI Panel:

To install the panel on the server, run the script:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will ask:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer `y` if you want to set the port yourself or `n`, and the script will assign a port automatically.
Avoid using default ports `22, 80, 8080`. It is recommended to choose another, such as `8181`.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the panel installation will be finished, and you can connect to it using a browser at:

```sh
http://YOUR_SERVER_IP_ADDRESS:PORT/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

When this address is entered, you will see the login window for the panel where you must input the username and password previously provided by the script.
