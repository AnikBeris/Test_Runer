## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform features compatibility with a wide range of architectures and devices, allowing flexibility in various computing environments. Below are the key supported architectures:

- **amd64:** A common architecture standard for personal computers and servers, ensuring smooth operation for most modern operating systems.

- **x86 / i386:** Widely used in desktops and laptops. The architecture is supported by many operating systems and applications, including `Windows, macOS, Linux`, among others.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices such as smartphones and tablets. Example devices include `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and more.

- **armv7 / arm / arm32:** Architecture for older mobile and embedded devices, still commonly found in devices such as `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and others.

- **armv6 / arm / arm32:** Focused on very old embedded devices. Though less common, it is still utilized in devices such as `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture primarily used in early embedded systems. Rare today, but applicable for outdated devices like early versions of `Raspberry Pi` and some older smartphones.

- **s390x:** Typically used in `IBM` mainframes, offering high performance and reliability for enterprise workloads.

</details>

## üìä Functions || Features

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

- System state monitoring
- Lookup for all incoming connections and clients
- Theme: `Dark / Light`
- Support for multi-user and multi-protocol
- Protocol support: `VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- Native XTLS-protocol support: `RPRX-Direct, Vision, REALITY`
- Traffic statistics, traffic limit, expiration limit
- Customizable `Xray` configuration templates
- Support for panel access via `HTTPS (custom domain + SSL certificate)`
- One-click request for an `SSL certificate` with automatic renewal
- Includes advanced configuration panel options
- Fixed `API routes` (user settings are created through the `API`)
- Support for configuration modifications via various parameters available in the panel
- Database export/import support through the panel

</details>

<h1 align="center"> 
SSL Certificate Cloudflare
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in function to request an `SSL certificate` through `Cloudflare`. To obtain a certificate using this script, you need:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- An email registered with `Cloudflare`
- Cloudflare Global API Key
- Domain name must be directed (set in DNS) to the current server via `Cloudflare`



## How to get the Global API Key from Cloudflare:

1. Run the command `x-ui` in the terminal, then select `Cloudflare SSL Certificate`.

2. Visit the link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. You might need to re-authenticate. After that, the key will be shown (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


When using, just enter your domain, `email`, and `API KEY`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

# 1. Quick Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-step Installation of 3X


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Required Packages

1. System Update
Before starting the installation, make sure your system is up to date. Run the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Installing Required Packages

Ensure that the necessary packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing 3X-UI Panel

To install the panel on your server, execute the script with the following command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will ask a question:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer `y` if you want to set the port yourself, or `n` if you want the script to assign a port automatically. 
Do not use standard ports like `22, 80, 8080`. We recommend choosing another port, for example, `8181`.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the panel installation will be finished, and you can connect to the panel via your browser at the following address:

```sh
http://YOUR_SERVER_IP_ADDRESS:port/WebBasePath (example: http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

Upon entering the address in your browser, you will reach the panel login page, where you need to input the username and password previously provided by the script.
