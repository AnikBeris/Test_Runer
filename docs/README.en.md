## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform ensures compatibility with a wide range of architectures and devices, providing flexibility in various computing environments. Below are the main architectures we support:

- **amd64:** This common architecture is standard for personal computers and servers, supporting seamless operation of most modern operating systems.

- **x86 / i386:** Widely used in desktop computers and laptops. The architecture is supported by numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices like smartphones and tablets. Example devices: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and more.

- **armv7 / arm / arm32:** Serves as the architecture for older mobile and embedded devices. Still widely used in devices like `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and others.

- **armv6 / arm / arm32:** Targeted at very old embedded devices. Although less common, it is still used in devices such as `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture primarily associated with early embedded systems. Rare today, but still found in legacy devices such as early `Raspberry Pi` versions and some old smartphones.

- **s390x:** This architecture is commonly used in `IBM` mainframes, providing high performance and reliability for enterprise workloads.

</details>

<h1 align="center"> 
SSL Certificate Cloudflare
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in feature to request an `SSL Certificate` through `Cloudflare`. To obtain the certificate using this script, you will need:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- Email registered with `Cloudflare`
- Global API Key Cloudflare
- Domain name must be pointed (registered in DNS) to the current server through `Cloudflare`



## How to obtain Global API Key Cloudflare:

1. In the terminal, execute the `x-ui` command, then choose `Cloudflare SSL Certificate`.

2. Go to the link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. Re-authentication may be required. Afterward, the key will be displayed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


When in use, simply enter your domain, `email`, and `API KEY`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

</details>

<h1 align="center"> 
Installing 3X-UI
</h1>


# 1. Quick Install 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-Step 3X Installation


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Required Packages

1. System Update
Before starting the installation, make sure your system is updated. Run the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Installing Required Packages

Ensure necessary packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing the 3X-UI Panel

To install the panel on your server, execute the following script command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During installation, the script will ask:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer `y` if you want to set the port yourself, or `n` for the script to assign it automatically.
Avoid using standard ports `22, 80, 8080`. We recommend choosing any other port, such as `8181`.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, installation of the panel is finished, and you can connect to the panel through your browser using the path:

```sh
http://YOUR_SERVER_IP:PORT/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

When you enter the address in your browser, you will see the login page for the panel, where you need to enter the username and password provided by the script earlier.
