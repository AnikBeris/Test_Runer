## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform offers compatibility with a wide range of architectures and devices, enabling flexibility across various computing environments. Below are key architectures supported:

- **amd64:** A common architecture standard for personal computers and servers providing seamless operation for most modern operating systems.

- **x86 / i386:** Widely utilized in desktops and laptops. The architecture is broadly supported across numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices such as smartphones and tablets. Examples include `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Serves as architecture for older mobile and embedded devices and is still universal in devices like `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, among others.

- **armv6 / arm / arm32:** Focused on very old embedded devices, it is less widespread but still applicable in `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture primarily tied to early embedded systems and rarely seen today, though possibly used in outdated devices such as the early `Raspberry Pi` versions.

- **s390x:** This architecture is typically used in IBM mainframes, offering high performance and reliability for enterprise workloads.

</details>

## üìä Functions || Features

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

- System state monitoring
- Search across all incoming connections and clients
- Theme: `Dark / Light`

- Supports changing configurations on various parameters available in the panel
- Database export/import through the panel

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt="Cloudflare" width="50%">
</div>

The management script includes a built-in feature for requesting an `SSL certificate` via `Cloudflare`. To obtain the certificate using this script, you will need:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- Email registered with `Cloudflare`
- Global API Key Cloudflare
- Domain name should be pointed (DNS-configured) to the current server via `Cloudflare`



## How to Obtain Global API Key Cloudflare:

1. In the terminal, execute the `x-ui` command and then select `Cloudflare SSL Certificate`.

2. Navigate to the link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt="API Key 1" width="70%">
</div>

4. You may need to authenticate again. Once authenticated, the key will be displayed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt="API Key 2" width="70%">
</div>

Simply enter your domain, `email`, and `API KEY`. Example shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt="Detail Enter" width="70%">
</div>

</details>

# 1. Fast Setup 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-Step Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Necessary Packages

1. System Update
Before starting the installation, ensure your system is updated by running the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Installing Required Packages

Verify that the necessary packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing 3X-UI Panel

To install the panel on your server, execute the script with the following command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During installation, the script will ask:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Reply with `y` if you wish to set the port manually, or `n` for the script to assign a port automatically. Avoid standard ports such as `22, 80, 8080`. We recommend selecting any other port, such as `8181`.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After following these steps, the panel installation will be complete, allowing connection via browser to the panel path:

```sh
http://server_IP_address:port/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

Enter the specified address in your browser to reach the login screen where you should input the username and password provided by the script.
