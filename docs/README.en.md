## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform offers compatibility with a wide range of architectures and devices to ensure flexibility across various computing environments. Below are the primary supported architectures:

- **amd64:** This popular architecture is the standard for personal computers and servers, ensuring seamless operation of most modern operating systems.

- **x86 / i386:** Widely used in desktops and laptops, this architecture enjoys broad support from numerous operating systems and applications, including, but not limited to, `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices such as smartphones and tablets. Examples of devices include: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Used for older mobile and embedded devices. Still widely used in devices like `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, among others.

- **armv6 / arm / arm32:** Geared towards very old embedded devices. Although less common, it is still utilized in examples like `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture primarily associated with early embedded systems. Rarely encountered today, it may be used in legacy devices, such as early versions of `Raspberry Pi` and some old smartphones.

- **s390x:** This architecture is typically employed in IBM mainframes and delivers high performance and reliability for enterprise workloads.

</details>

<h1 align="center"> 
SSL Certificate from Cloudflare
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in feature to request an `SSL certificate` via `Cloudflare`. To obtain a certificate using this script, you will need:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- An email registered with `Cloudflare`
- Cloudflare Global API Key
- The domain name must be routed (set in DNS) to the current server via `Cloudflare`



## How to obtain the Cloudflare Global API Key:

1. In the terminal, execute the command `x-ui`, then select `Cloudflare SSL Certificate`.

2. Visit the link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click on `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. Reauthentication may be required. After that, the key will be displayed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


When using it, simply enter your domain, `email`, and `API KEY`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

</details>

<h1 align="center"> 
Installing 3X-UI
</h1>


# 1. Quick Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-Step Installation of 3X


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Required Packages

1. System Update
Before starting the installation, ensure your system is updated. Run the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Installing necessary packages

Make sure the required packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing the 3X-UI Panel

To install the panel on the server, execute the script with the following command:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will prompt:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Reply `y` if you wish to set the port manually, or `n` to let the script set the port automatically.
Do not use standard ports such as `22, 80, 8080`. We recommend choosing another port like `8181`.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the panel installation will be finished, and you can connect to the panel via the browser by going to:

```sh
http://your_server_IP:port/WebBasePath (example: http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

Once you enter the address in the browser, you will see the login window for the panel. Enter the username and password that the script provided earlier.
