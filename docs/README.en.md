## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Details</summary>

Our platform offers compatibility with a wide range of architectures and devices, providing flexibility across various computing environments. The primary supported architectures are listed below:

- **amd64:** A common architecture standard for personal computers and servers, ensuring seamless operation of the majority of modern operating systems.
- **x86 / i386:** Widely implemented in desktop computers and laptops. This architecture has extensive support from numerous OSs and applications, including `Windows, macOS, and Linux`.
- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices such as smartphones and tablets. Examples: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and more.
- **armv7 / arm / arm32:** Architecture for older mobile and embedded devices, still extensively used in devices such as `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, etc.
- **armv6 / arm / arm32:** Focused on very old embedded devices. While less common, it is still used, e.g., in `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.
- **armv5 / arm / arm32:** An older architecture, primarily associated with early embedded systems. Rare today, but finds application in obsolete devices such as early `Raspberry Pi` versions and some old smartphones.
- **s390x:** Typically used in IBM mainframes and delivers high performance and reliability for enterprise workloads.

</details>

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in function for requesting an `SSL certificate` via `Cloudflare`. To obtain a certificate using this script, you need:

<details> 
    <summary>‚öôÔ∏è Expand Details </summary>

- Email registered with `Cloudflare`
- Global API Key from Cloudflare
- The domain name must be directed (specified in DNS) to the current server via `Cloudflare`

## How to obtain the Global API Key from Cloudflare:

1. In the terminal, run the command `x-ui`, then select `Cloudflare SSL Certificate`.

2. Go to the link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click on `View Global API Key` (see the screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. Re-authentication might be required. Afterward, the key will be displayed (see the screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>

To use it, simply enter your domain, `email`, and `API KEY`. An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

<h1 align="center"> 
Installation of 3X-UI
</h1>

# 1. Quick Installation of 3X

<details> 
    <summary>‚öôÔ∏è Expand Details </summary>

```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh
```

</details>

# 2. Step-by-Step Installation of 3X

<details> 
    <summary>‚öôÔ∏è Expand Details </summary>

# üìä Installation of Required Packages

1. System Update
Before starting the installation, ensure your system is updated. Execute the commands below:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Install Necessary Packages

Ensure the required packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```

## Installation of the 3X-UI Panel

Run the script command to install the panel on the server:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will ask a question:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer `y` if you'd like to set the port yourself, or `n` if you'd like the script to randomly assign a port.
Avoid default ports such as `22, 80, 8080`. It's recommended to choose any other port, like `8181`.

```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the installation of the panel will be done, and you can access the panel via a web browser at:

```sh
http://Server_IP_Address:Port/WebBasePath (e.g., http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

After entering the address in the browser, you will see the panel login screen where you need to enter the login and password generated during the script installation.
