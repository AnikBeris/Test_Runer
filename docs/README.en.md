## üìä Supported Architectures and Devices

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

Our platform offers compatibility with a broad range of architectures and devices, ensuring flexibility in various computing environments. Below are the main architectures we support:

- **amd64:** This common architecture serves as the standard for personal computers and servers, ensuring seamless performance across most modern operating systems.

- **x86 / i386:** Widely used in desktop computers and laptops, this architecture is supported by numerous operating systems and applications, including `Windows, macOS, and Linux`.

- **armv8 / arm64 / aarch64:** Designed for modern mobile and embedded devices such as smartphones and tablets. Example devices include: `Raspberry Pi 4, Raspberry Pi 3, Raspberry Pi Zero 2/Zero 2 W`, `Orange Pi 3 LTS`, and others.

- **armv7 / arm / arm32:** Serves as the architecture for more dated mobile and embedded devices. Still widely used in devices such as `Orange Pi Zero LTS, Orange Pi PC Plus, Raspberry Pi 2`, and others.

- **armv6 / arm / arm32:** Geared towards very old embedded devices. Although less common, it is still seen in devices like the `Raspberry Pi 1, Raspberry Pi Zero/Zero W`.

- **armv5 / arm / arm32:** An older architecture typically associated with early embedded systems. Still occasionally encountered in outdated devices like early `Raspberry Pi` versions and some old smartphones.

- **s390x:** This architecture is typically found in `IBM` mainframes and provides high performance and reliability for enterprise workloads.

</details>

## üìä Features

<details> 
    <summary>‚öôÔ∏è Expand Description</summary>

- System monitoring
- Search function for all incoming connections and clients
- Themes: `dark / light`
- Multi-user and multi-protocol support
- Supported protocols: `VMESS, VLESS, Trojan, Shadowsocks, Dokodemo-door, Socks, HTTP, WireGuard`
- Native support for XTLS protocols: `RPRX-Direct, Vision, REALITY`
- Traffic statistics, traffic limit, expiration time restriction
- Customizable `Xray` configuration templates

- One-click `SSL certificate` request and auto renewal
- For advanced configuration options, see the panel
- Fixed `API routes` (user settings are created through `API`)
- Support for changing configurations via different parameters available in the panel
- Support for importing/exporting the database through the panel  

</details>

<h1 align="center"> 
Cloudflare SSL Certificate
</h1>

<div align="center">
  <img src="../media/Tutorial/Article_1/cloudflare.jpg" alt=" Cloud flare " width="50%">
</div>

The management script includes a built-in function to request an `SSL certificate` via `Cloudflare.` To obtain the certificate using this script, you need:


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

- Email registered with `Cloudflare`
- Global API Key Cloudflare
- Domain name must be pointed (DNS configured) to the current server via `Cloudflare`



## How to get a Global API Key from Cloudflare:

1. In the terminal, execute the command `x-ui` and then select `Cloudflare SSL Certificate`.

2. Follow this link: [Cloudflare API Tokens](https://dash.cloudflare.com/profile/api-tokens)

3. Click `View Global API Key` (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey1.png" alt=" API Key 1 " width="70%">
</div>

4. You may need to reauthenticate. Afterward, the key will be displayed (see screenshot below):

<div align="center">
  <img src="../media/Tutorial/Article_1/APIKey2.png" alt=" API Key 2 " width="70%">
</div>


When using, simply enter your domain, `email`, and `API KEY.` An example is shown below:

<div align="center">
  <img src="../media/Tutorial/Article_1/DetailEnter.png" alt=" Detail Enter " width="70%">
</div>

# 1. Quick Installation of 3X

<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>


```sh
sudo apt update && sudo apt upgrade -y && \
sudo apt install -y git curl openssl qrencode systemd && \
rm -rf self_signed_certificate.sh && \
curl -O https://raw.githubusercontent.com/AnikBeris/self-signed-certificate/main/self_signed_certificate.sh && \
chmod +x self_signed_certificate.sh && \
bash ./self_signed_certificate.sh

```

</details> 

# 2. Step-by-Step Installation of 3X


<details> 
    <summary> ‚öôÔ∏è Expand Description </summary>

# üìä Installing Necessary Packages

1. Update the system
Before beginning the installation, ensure your system is up-to-date. Run the following commands:

```sh
sudo apt update && sudo apt upgrade -y &&
```

2. Install necessary packages

Make sure the required packages are installed on your server:

```sh
apt-get install wget curl openssl qrencode systemd -y
```



## Installing the 3X-UI Panel

To install the panel on the server, run the script:

```sh
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

During the installation, the script will prompt:

```sh
Would you like to customize the Panel Port settings? (If not, a random port will be applied) [y/n]:
```

Answer with `y` if you want to set the port yourself, or `n` to let the script assign a port automatically.
Avoid using standard ports like `22, 80, 8080`. We recommend selecting another, such as `8181`.


```sh
Username: 3favnjd8

Password: Msdf823Ll

Port: 40608

WebBasePath: vpkPI6ex9ajesDX

Access URL: http://192.168.0.149:40608/vpkPI6ex9ajesDX
```

After completing these steps, the panel installation will be complete. You can access the panel in your browser by navigating to:

```sh
http://Server_IP_Address:Port/WebBasePath (example: http://192.168.0.10:40608/vpkPI6ex9ajesDX)
```

After entering the address in your browser, you'll be taken to the panel login window where you need to enter the username and password provided by the script earlier.
